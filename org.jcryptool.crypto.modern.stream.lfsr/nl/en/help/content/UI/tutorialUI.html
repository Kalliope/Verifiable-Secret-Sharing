LFSR

This tutorial will guide you through an encryption and a decryption with a modern stream cipher, the Linear Feedback Shift Register (LFSR), which is provided by the org.jcryptool.crypto.modern.stream.lfsr plug-in.
Note: As with all algorithms, you can only perform the following steps when you have an opened editor window.

Select the size of the LFSR (restricted to a value between 1 and 25 for presentation purposes). You then have to select which bits are tapped and the value of the seed (the LFSR's initial state). In our example we have an LFSR of length 21. The 5th, 14th, 17th, 18th and 21st bits are tapped (note that the final bit is always considered to be tapped). The LFSR's seed is 100100100010001000110. Clicking on the Finish button will start the encryption/ decryption process.

The resulting ciphertext (in output001.bin) can be decrypted, by selecting identical settings and values in the same wizard that was used for encryption. (Note: The output is opened in a hex editor rather than a text editor. This is because modern algorithms are based on bytes, not characters; as such, opening a text editor would be inappropriate.)

Clicking on the Finish button will now start the decryption of the ciphertext and yield the correct plaintext which will be displayed in a new editor (output002.bin in the screenshot below).

Outputting the keystream

Selecting the Display output file with keystream in a separate hex editor radio button will open a second file containing the generated keystream from that encryption/ decryption process. A comparison of the keystream from the encryption process to that of the decryption process will show that given identical inputs, an identical keystream is produced.

The keystream is displayed in a new editor (output004.bin above), with the plaintext being displayed as usual (output003.bin).
Selecting the Display keystream only (of specified number of bytes): radio button will open a file containing a specified number of bytes of generated keystream. The number of bytes of keystream to generate is specified in a textbox which is enabled when the Display keystream only button is selected.

Our example shows the keystream generated, given the same seed and tap settings input as earlier, up to 100 bytes.
